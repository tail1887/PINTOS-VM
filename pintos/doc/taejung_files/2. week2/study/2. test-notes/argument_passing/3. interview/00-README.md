# 3. interview — 면접 대비 자료

이 폴더는 Argument Passing 구현을 면접에서 설명할 때 사용하는 대본과 팁을 담고 있습니다.

## 사용 가이드

### 준비 방법
1. `00-interview-script.md`를 먼저 읽고 전체 흐름 파악
2. Q1-Q10을 순서대로 음독해보기
3. 각 답변을 자신의 경험에 맞춰 커스터마이징

### 추천 학습 순서
- **첫 번째 시도**: 전체 대본을 빠르게 읽기 (5-10분)
- **두 번째 시도**: 각 Q/A마다 5-10초 멈추고 자신 말로 재설명
- **세 번째 시도**: 30초 요약본만 보고 전체 흐름 설명
- **최종**: 누군가에게 실제로 설명해보기

### 실제 면접 팁
- **파트 1 (0-2분)**: Q1의 30초 버전 + Q2 정도로 시작
- **파트 2 (2-5분)**: Q3의 스택 레이아웃 시각화하며 설명
- **파트 3 (5-10분)**: Q5(어려움), Q7(버그)로 깊이감 드러내기
- **마무리**: Q8(실제 OS와의 연결)로 마무리하면 좋은 인상

### 이 대본이 커버하는 주제

| Q | 주제 | 면접 가치 |
|---|------|---------|
| Q1 | 기본 개념 | 🌟 필수 (30초 명확성) |
| Q2 | 3단계 흐름 | 🌟 필수 (구조 이해도) |
| Q3 | 스택 레이아웃 | 🌟⭐ 가장 중요 (실행 능력) |
| Q4 | 정렬의 이유 | 🌟 필수 (깊이있는 이해) |
| Q5 | 어려움 극복 | 🌟⭐ 중요 (문제해결력) |
| Q6 | 테스트 전략 | 🌟 필수 (체계적 접근) |
| Q7 | 버그 경험 | 🌟⭐ 중요 (자기반성) |
| Q8 | 실용성 | 🌟 필수 (시야 확장) |
| Q9 | 심화 개념 | ⭐ 좋음 (깊이있는 답변) |
| Q10 | 코드 리뷰 | 🌟⭐ 중요 (엔지니어 마인드) |

## 꼭 기억할 3가지

1. **"왜"를 설명하기**: 단순히 스택에 올린다X → 왜 역순으로 올리는지O
2. **구체적 예시**: 추상적 설명X → "echo hello world 예시"O
3. **경계 케이스**: 행복한 경로만X → 실패 경로/엣지 케이스도 언급O

# 05 — `args-dbl-space` 검증

## 1. 이 테스트가 검증하는 것
공백이 여러 개 연속되는 입력에서
파싱 결과가 단일 공백과 동일하게 처리되는지 검증합니다.

## 2. 구현 필요 함수와 규칙
### 토큰화 루틴
- 위치: `pintos/userprog/process.c`
- 규칙 1: 연속 공백은 토큰 구분자 하나로 취급한다.
- 규칙 2: 빈 문자열 토큰을 만들지 않는다.

### argc/argv 확정 단계
- 위치: `pintos/userprog/process.c`
- 규칙 1: 실제 토큰 개수만 `argc`로 계산한다.
- 규칙 2: `argv` 순서는 공백 개수와 무관하게 동일하다.

## 3. 실패 시 바로 확인할 것
- 빈 토큰이 포함되어 `argc`가 과대 계산되는지
- 문자열 끝의 공백/앞 공백 처리에서 인덱스 오프셋 오류가 있는지

## 4. 재검증
- `args-dbl-space` 단일 실행 통과 확인

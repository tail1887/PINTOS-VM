# 02 — `args-single` 검증

## 1. 이 테스트가 검증하는 것
단일 인자 전달에서 `argc`, `argv[0]`, `argv[1]`이 정확한지 검증합니다.

## 2. 구현 필요 함수와 규칙
### `process_exec()` 토큰화 단계
- 위치: `pintos/userprog/process.c`
- 규칙 1: 첫 토큰은 프로그램 이름, 다음 토큰은 첫 인자로 유지한다.

### 사용자 스택 구성 단계
- 위치: `pintos/userprog/process.c`
- 규칙 1: 문자열 복사 후 주소를 포인터 배열에 정확히 매핑한다.
- 규칙 2: `argv[2] == NULL`을 보장한다.

## 3. 실패 시 바로 확인할 것
- 문자열 복사 순서(역순 push)와 포인터 순서(오른쪽->왼쪽 push)가 섞였는지
- `RDI/RSI` 레지스터 설정 오류가 없는지

## 4. 재검증
- `args-single` 단일 실행 통과 확인

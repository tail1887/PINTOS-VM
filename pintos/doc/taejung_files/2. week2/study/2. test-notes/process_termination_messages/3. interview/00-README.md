# 3. interview — Process Termination Messages

이 폴더는 종료 메시지 요구사항을 면접에서 설명하기 위한 자료입니다.

## 핵심 답변
종료 메시지는 단순 출력 기능이 아니라 프로세스 종료 공통 경로를 제대로 잡았는지 보여주는 요구사항입니다. `exit(status)`뿐 아니라 사용자 프로세스가 예외로 죽는 경우도 같은 종료 경로로 들어오게 만들고, `process_exit()`에서 정해진 형식으로 한 번만 출력하게 설계합니다.

## 꼭 말할 포인트
- 출력 형식은 채점 스크립트와 직접 연결되므로 한 글자도 다르면 안 된다.
- status는 `sys_exit()` 또는 exception kill 경로에서 먼저 저장한다.
- 메시지는 공통 종료 경로에서 한 번만 출력해야 중복을 피할 수 있다.
- 커널 스레드와 halt syscall에는 출력하지 않는다.

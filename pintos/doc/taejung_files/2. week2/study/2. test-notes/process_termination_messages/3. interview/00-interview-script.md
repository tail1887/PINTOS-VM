# Process Termination Messages 면접 대본

## Q1. 종료 메시지는 왜 필요한가요?
PintOS 테스트가 사용자 프로세스의 종료를 로그로 확인하기 때문입니다. 그래서 프로세스가 종료될 때 `name: exit(status)` 형식을 정확히 맞춰 출력해야 합니다.

## Q2. 어디에서 출력하는 게 좋은가요?
`sys_exit()`에서만 출력하면 page fault 같은 비정상 종료를 놓칠 수 있습니다. 그래서 exit status는 각 종료 원인에서 저장하고, 출력은 사용자 프로세스 종료 공통 경로인 `process_exit()`에서 한 번만 하는 방식이 안정적입니다.

## Q3. 주의할 점은 무엇인가요?
커널 스레드 종료와 halt syscall에는 출력하면 안 됩니다. 또 디버그 출력이 섞이면 채점 스크립트가 실패할 수 있으므로 정해진 메시지 외에는 출력하지 않아야 합니다.

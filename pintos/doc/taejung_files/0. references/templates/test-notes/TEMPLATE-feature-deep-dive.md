## 5. 구현 주석 (위치별 정리)

이 장은 **이 문서 주제 안에서 실제로 손대는 함수·구조체·분기·실패 경로마다** 구현 주석을 둔다.
4장의 소기능(4.1, 4.2, …)마다 대응하는 `5.N`이 있어야 한다.
구현자가 이 장만 보고 "어느 파일의 어느 함수에서 무엇을 추가해야 하는지" 바로 알 수 있어야 한다.
`01` 번대 **개요 문서**만 예외로, 큰 그림 전용이면 5장을 생략할 수 있되, 대신 하위 feature 문서의 5장으로 안내한다.
규칙과 체크 순서는 아래 기준을 반드시 지킨다.
- `규칙`: 반드시 지켜야 하는 상태/계약(불변식)만 적는다. 구체 코드 줄(`if`, `malloc`, `ASSERT`, 함수 호출 나열)을 그대로 쓰지 않는다.
- `구현 체크 순서`: 실제 구현 수행 순서를 번호대로 적는다. 구현자가 그대로 따라가며 코드 작성/점검할 수 있어야 한다.

# Getting Started

## 1) 시작 환경

시작하려면 먼저 Pintos를 빌드할 수 있는 머신에 로그인해야 합니다.

- 사용 가능: 교수진이 제공한 Linux 머신, 개인 Solaris/Linux 머신
- 주의: 채점은 **제공된 머신 환경**에서 진행됩니다.
- 안내 기준: 이 문서는 제공된 환경을 전제로 작성되었습니다.
- 제한: 개인 머신 설치/운영 관련 별도 지원은 제공되지 않습니다.

## 2) 소스 코드 받기

아래 명령으로 `pintos` 디렉터리에 소스 코드를 내려받을 수 있습니다.

```bash
git clone https://github.com/casys-kaist/pintos-kaist
```

프로젝트 시작 전에는 템플릿 소스가 바뀔 수 있으므로, 레포지토리 복제를 권장합니다.  
템플릿이 변경되면 이 방식으로 쉽게 동기화할 수 있습니다.

> 공개 포크(public fork)는 만들지 마세요.  
> 프로젝트 기간 중에는 템플릿 코드의 치명적 버그를 제외하고 소스 코드가 변경되지 않습니다.

## 3) 검증된 기준 환경

교수진은 아래 환경(제공 머신과 동일)에서 Pintos 코드와 정답 코드를 테스트했습니다.

- gcc: `gcc (Ubuntu 7.4.0-1ubuntu1~16.04~ppa1) 7.4.0`
- qemu-system-x86_64: `QEMU emulator version 2.5.0 (Debian 1:2.5+dfsg-5ubuntu10.43)`

## 4) 초기 환경 설정

소스 코드를 받은 뒤 Pintos 루트 디렉터리에서 아래 명령을 실행합니다.

```bash
source ./activate
```

이 줄은 홈 디렉터리의 `.bashrc`에 추가해 두는 것을 권장합니다.  
그렇지 않으면 로그인할 때마다 매번 직접 입력해야 합니다.

## 5) 소스 트리 개요

`pintos/`에서 확인해야 할 주요 디렉터리는 다음과 같습니다.

- `threads/`: 기본 커널 소스. Project 1부터 수정
- `userprog/`: 사용자 프로그램 로더 소스. Project 2부터 수정
- `vm/`: Project 3에서 가상 메모리를 구현할 디렉터리
- `filesys/`: 기본 파일시스템 소스. Project 2부터 사용, Project 4 전까지는 미수정
- `devices/`: 키보드/타이머/디스크 등 I/O 코드. Project 1에서 타이머 구현 수정
- `lib/`: 표준 C 라이브러리 일부 구현. 커널 및(2주차 이후) 사용자 프로그램에 함께 컴파일
- `include/lib/kernel/`: 커널 전용 라이브러리/자료구조(비트맵, 이중 연결 리스트, 해시 테이블)
- `include/lib/user/`: 사용자 프로그램 전용 라이브러리
- `tests/`: 프로젝트 테스트 코드 (채점 전 원본으로 교체됨)
- `examples/`: Project 2부터 사용하는 사용자 프로그램 예제
- `include/`: 헤더 파일(`*.h`) 소스

## 6) Pintos 빌드

첫 프로젝트 소스를 빌드하려면:

1. `threads` 디렉터리로 이동
2. `make` 실행

그러면 `threads/build`가 생성되고, 그 안에서 커널이 빌드됩니다.

### build 디렉터리 핵심 파일

- `Makefile`: `pintos/src/Makefile.build` 복사본. 커널 빌드 규칙 정의
- `kernel.o`: 커널 전체 오브젝트 파일(디버그 정보 포함)
- `kernel.bin`: 디버그 정보를 제거한 커널 메모리 이미지(512kB 제한 대응)
- `loader.bin`: 커널 로더 메모리 이미지(512바이트 고정, BIOS 제약)

또한 `build` 하위에는 컴파일 산출물(`.o`)과 의존성 파일(`.d`)이 생성됩니다.  
의존성 파일은 어떤 파일을 재컴파일해야 하는지 `make`에 알려줍니다.

## 7) Pintos 실행

시뮬레이터 실행용 `pintos` 프로그램이 제공됩니다.  
기본 형태는 `pintos argument...`이며, 인자는 커널로 전달됩니다.

예시:

```bash
pintos run alarm-multiple
```

이 명령은 커널에 `run alarm-multiple`을 전달해 해당 테스트를 실행합니다.

직렬 출력은 파일로 저장할 수 있습니다.

```bash
pintos -- run alarm-multiple > logfile
```

## 8) 실행 옵션

`pintos`는 QEMU/가상 하드웨어 설정 옵션을 지원합니다.

- 옵션은 커널 인자보다 앞에 둡니다.
- 옵션과 커널 인자는 `--`로 구분합니다.
- 형태: `pintos option... -- argument...`

대표 옵션:

- `-v`: VGA 출력 비활성화
- `-s`: stdin 직렬 입력 및 stdout 출력 억제

사용 가능한 전체 옵션은 인자 없이 `pintos`를 실행해 확인할 수 있습니다.  
커널 명령/옵션 목록은 `pintos -h`로 확인 가능합니다.

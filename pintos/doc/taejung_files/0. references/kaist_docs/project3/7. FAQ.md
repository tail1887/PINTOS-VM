### 페이지 폴트를 처리한 뒤에는 프로세스를 어떻게 재개하나요?

`page_fault()`에서 반환하면 현재 사용자 프로세스가 재개됩니다. 그러면 instruction pointer가 가리키는 명령을 다시 시도합니다. 사용자 프로세스가 왜 가끔 스택 포인터보다 위쪽에서 폴트를 내는지 궁금할 수 있습니다. stack growth 테스트에서 사용자 프로그램이 현재 사용자 프로그램의 스택 포인터보다 위쪽 주소에서 폴트를 내는 것을 볼 수 있습니다.

### 가상 메모리 시스템이 데이터 세그먼트 증가를 지원해야 하나요?

아니요. 데이터 세그먼트의 크기는 링커가 결정합니다. Pintos에는 여전히 동적 할당이 없습니다. 다만 memory-mapped file을 이용해 사용자 수준에서 이를 "흉내"내는 것은 가능합니다. 잘 설계된 시스템이라면 데이터 세그먼트 증가 지원은 약간의 추가 복잡도만 더할 것입니다.

### 페이지 프레임 할당에 왜 `PAL_USER`를 사용해야 하나요?

`palloc_get_page()`에 `PAL_USER`를 전달하면 메인 커널 풀 대신 사용자 풀에서 메모리를 할당합니다. 사용자 풀의 페이지가 고갈되면 사용자 프로그램이 page 처리될 뿐이지만, 커널 풀의 페이지가 고갈되면 많은 커널 함수가 메모리를 얻어야 하므로 여러 실패가 발생합니다. 원한다면 `palloc_get_page()` 위에 다른 allocator를 얹어도 되지만, 기반 메커니즘은 이것이어야 합니다. 또한 `-ul` 커널 커맨드라인 옵션을 사용해 사용자 풀의 크기를 제한할 수 있으며, 이를 통해 다양한 사용자 메모리 크기에서 VM 구현을 쉽게 테스트할 수 있습니다.

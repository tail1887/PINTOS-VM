Memory swapping은 물리 메모리 사용을 극대화하기 위한 메모리 회수 기법입니다. 메인 메모리의 프레임이 모두 할당되면, 시스템은 사용자 프로그램의 추가 메모리 할당 요청을 더 이상 처리할 수 없습니다. 한 가지 해결책은 현재 사용 중이 아닌 메모리 프레임을 디스크로 swap out하는 것입니다. 이렇게 하면 일부 메모리 자원이 해제되어 다른 애플리케이션에서 사용할 수 있게 됩니다.

Swapping은 운영체제가 수행합니다. 시스템이 메모리가 부족한 상태에서 메모리 할당 요청을 받으면, swap disk로 내보낼 페이지를 선택합니다. 그런 다음 메모리 프레임의 정확한 상태를 디스크에 복사합니다. 프로세스가 swapped out된 페이지에 접근하려고 하면, OS는 정확한 내용을 다시 메모리로 가져와 해당 페이지를 복구합니다.

Eviction 대상으로 선택되는 페이지는 익명 페이지일 수도 있고 file-backed page일 수도 있습니다. 이 섹션에서는 각 경우를 처리합니다.

모든 swapping 작업은 명시적으로 호출되지 않고 function pointer를 통해 호출됩니다. 이 함수 포인터들은 `struct page_operations file_ops`의 멤버이며, 각 페이지 initializer의 operation으로 등록됩니다.

### 익명 페이지

**`vm/anon.c`의 `vm_anon_init`과 `anon_initializer`를 수정하세요.** 익명 페이지에는 backing storage가 없습니다. 익명 페이지의 swapping을 지원하기 위해, 임시 backing storage인 swap disk를 제공합니다. 여러분은 swap disk를 사용해 익명 페이지의 swap을 구현해야 합니다.

---

```
void vm_anon_init (void);
```

> 이 함수에서 swap disk를 설정해야 합니다. 또한 swap disk 안의 free 영역과 used 영역을 관리할 자료구조도 필요합니다. swap 영역도 PGSIZE, 즉 4096바이트 단위로 관리됩니다.

---

```
bool anon_initializer (struct page *page, enum vm_type type, void *kva);
```

> 익명 페이지의 initializer입니다. Swapping을 지원하기 위해 `anon_page`에 몇 가지 정보를 추가해야 합니다.

이제 익명 페이지의 swapping을 지원하기 위해 `vm/anon.c`의 `anon_swap_in`과 `anon_swap_out`을 구현하세요. 어떤 페이지가 swap in되려면 먼저 swap out되어야 하므로, `anon_swap_in`을 구현하기 전에 `anon_swap_out`을 먼저 구현하는 것이 좋을 수 있습니다. 데이터 내용을 swap disk로 옮기고, 다시 메모리로 안전하게 가져와야 합니다.

---

```
static bool anon_swap_in (struct page *page, void *kva);
```

> swap disk에서 데이터 내용을 읽어 메모리로 가져와 익명 페이지를 swap in합니다. 데이터의 위치는 페이지가 swap out될 때 page struct에 저장되어 있어야 합니다. swap table을 갱신하는 것을 잊지 마세요. [Managing the Swap Table](https://casys-kaist.github.io/pintos-kaist/project3/introduction.html)을 참고하세요.

---

```
static bool anon_swap_out (struct page *page);
```

> 메모리의 내용을 디스크로 복사해 익명 페이지를 swap out합니다. 먼저 swap table을 사용해 디스크에서 빈 swap slot을 찾은 다음, 페이지의 데이터를 해당 slot에 복사합니다. 데이터 위치는 page struct에 저장되어야 합니다. 디스크에 더 이상 빈 slot이 없다면 커널을 panic시킬 수 있습니다.

### File-Mapped Page

file-backed page의 내용은 파일에서 오므로, mmap된 파일을 backing store로 사용해야 합니다. 즉 file-backed page를 evict하면 해당 페이지가 매핑된 원본 파일에 다시 기록됩니다. `vm/file.c`의 `file_backed_swap_in`, `file_backed_swap_out`을 구현하세요. 설계에 따라 `file_backed_init`과 `file_initializer`를 수정할 수도 있습니다.

---

```
static bool file_backed_swap_in (struct page *page, void *kva);
```

> 파일에서 내용을 읽어 `kva`에 있는 페이지를 swap in합니다. 파일 시스템과 동기화해야 합니다.

---

```
static bool file_backed_swap_out (struct page *page);
```

> 페이지의 내용을 파일에 다시 기록해 swap out합니다. 먼저 페이지가 dirty인지 확인하는 것이 좋습니다. dirty가 아니라면 파일의 내용을 수정할 필요가 없습니다. 페이지를 swap out한 뒤에는 해당 페이지의 dirty bit를 끄는 것을 잊지 마세요.

# Alarm Clock

`devices/timer.c`에 정의된 `timer_sleep()`을 재구현하세요.

현재도 동작하는 구현이 제공되지만 busy waiting 방식입니다. 즉, 충분한 시간이 지날 때까지 현재 시간을 반복 확인하고 `thread_yield()`를 계속 호출합니다.  
이 과제에서는 busy waiting을 피하도록 다시 구현해야 합니다.

```c
void timer_sleep (int64_t ticks);
```

이 함수는 호출한 스레드를 최소 `x` timer tick이 경과할 때까지 잠재웁니다.  
시스템이 유휴 상태가 아닌 한, 정확히 `x` tick 시점에 깨어날 필요는 없습니다. 올바른 시간만큼 기다린 뒤 ready queue에 넣으면 됩니다.

`timer_sleep()`은 실시간 성격의 스레드(예: 1초마다 커서 깜빡임)에 유용합니다.  
인자는 밀리초 같은 단위가 아니라 **timer tick 단위**입니다. 1초당 tick 수는 `TIMER_FREQ`이며, `devices/timer.h`에 정의됩니다. 기본값은 `100`입니다.

> `TIMER_FREQ` 값을 바꾸는 것은 권장하지 않습니다.  
> 값을 변경하면 많은 테스트가 실패할 가능성이 큽니다.

밀리초/마이크로초/나노초 단위 sleep 함수인 `timer_msleep()`, `timer_usleep()`, `timer_nsleep()`도 존재하지만, 필요 시 내부에서 `timer_sleep()`을 자동 호출합니다.  
따라서 이 함수들은 수정할 필요가 없습니다.

Alarm Clock 구현은 이후 프로젝트 필수는 아니지만, Project 4에서 도움이 될 수 있습니다.

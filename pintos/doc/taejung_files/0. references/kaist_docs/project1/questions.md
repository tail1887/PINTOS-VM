### [학습 요약: 운영체제 및 Pintos 핵심 개념]

본 문서는 동시성, 인터럽트, 스케줄링 및 운영체제 동기화 메커니즘에 대한 질의응답 정리본입니다.

#### 1. 컴퓨터 시스템의 기초와 동시성
* **동기화(Synchronization):** 여러 스레드가 공유 자원에 접근할 때 데이터 일관성을 유지하기 위한 순서 제어 및 접근 통제.
* **동시성(Concurrency) vs 병렬성(Parallelism):**
    * 동시성: CPU를 짧은 시간 단위로 나누어 여러 작업을 번갈아 실행(시분할).
    * 병렬성: 멀티코어 환경에서 물리적으로 여러 명령어를 동시에 수행.
* **레이스 컨디션(Race Condition):** 공유 자원에 동시 접근할 때 실행 순서에 따라 결과가 달라지는 오류 상태.

#### 2. 인터럽트와 스케줄링
* **인터럽트(Interrupt):** CPU가 하던 일을 멈추고 긴급한 하드웨어/소프트웨어 요청을 먼저 처리하도록 알리는 신호. 인터럽트를 끄면 스케줄링이 정지되어 동시성이 사라짐.
* **컨텍스트 스위칭(Context Switching):** 프로세스나 스레드 전환 시 레지스터, PC 등 상태 정보를 교체하는 과정.
* **스케줄러 vs 핸들러:**
    * **인터럽트 핸들러:** 하드웨어 이벤트에 대응하여 스케줄링을 '예약'하는 트리거(비서).
    * **스케줄러:** 우선순위를 분석하여 실제 스레드를 선택하고 교체하는 집행자(전략가).

#### 3. 동기화 프리미티브와 구현 기법
* **Lock/Mutex:** 한 번에 하나의 스레드만 공유 자원에 접근하도록 제한.
* **Semaphore:** 카운터를 사용하여 자원의 접근 수량 조절 및 순서 제어.
* **Condition Variable:** 특정 조건이 만족될 때까지 스레드를 대기(Block)시키고 깨우는 통신 장치.
* **인터럽트 비활성화(Interrupt Disabling):** 커널 수준에서 레이스 컨디션을 방지하기 위한 최후의 동기화 수단. 핸들러와 스레드 간의 데이터 보호에 필수적.
* **최적화 배리어(Optimization Barrier):** 컴파일러에 의한 명령어 재배치를 방지하여 하드웨어 상태 변화를 확실히 보장.

#### 4. Pintos 실습 개념
* **Advanced Scheduler (MLFQS):** CPU 사용량과 시스템 부하(`load_avg`)에 기반하여 동적으로 우선순위를 결정하는 피드백 루프 방식.
* **Tics:** 운영체제 시간 관리의 최소 단위(심장박동).

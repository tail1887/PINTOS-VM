# Grading

과제 평가는 **테스트 결과**를 기준으로 진행됩니다.

## Testing

테스트 점수는 교수진이 실행하는 테스트 결과를 기반으로 부여됩니다.  
각 프로젝트에는 여러 테스트가 있고, 이름은 `tests`로 시작합니다.

### 전체 테스트 실행

프로젝트 `build` 디렉터리에서 아래 명령을 실행하세요.

```bash
make check
```

이 명령은:

- 각 테스트를 빌드/실행
- 테스트별 pass/fail 출력
- 실패 시 상세 원인 출력
- 전체 테스트 종료 후 요약 결과 출력

### 단일 테스트 실행

테스트 `t`는 아래 흐름으로 채점됩니다.

- 출력 저장: `t.output`
- 스크립트 판정: pass/fail
- 결과 저장: `t.result`

단일 테스트 예시:

```bash
make tests/threads/alarm-multiple.result
```

`make`가 결과가 최신(up-to-date)이라고 표시해도 다시 실행하려면:

- `make clean` 실행 또는
- 해당 `.output` 파일 수동 삭제

### 상세 출력(Verbose)

기본적으로 테스트는 실행 중간 출력 없이 완료 시점에만 피드백을 제공합니다.  
진행 과정을 보고 싶다면:

```bash
make check VERBOSE=1
```

### 테스트 코드 수정 관련 주의

- 테스트 및 관련 파일 위치: `pintos/src/tests`
- 디버깅을 위해 일부 테스트를 수정하는 것은 가능
- 단, 채점 전에는 해당 디렉터리를 **원본으로 교체**하여 채점

즉, 실제 점수는 항상 원본 테스트 기준입니다.

### 공정성 및 일반성

- 테스트 자체 결함이 의심되면 제보하세요. 검토 후 필요 시 수정합니다.
- 제공된 테스트셋을 악용하는 방식은 허용되지 않습니다.
- 코드는 특정 테스트 이름에 의존하지 않고 **일반적인 경우**에도 올바르게 동작해야 합니다.
- 경계선에 있는 해결 방식이라면 사전에 문의하세요.

## Submission

아카이브 파일 생성 명령이 제공됩니다.  
`threads`, `userprog`, `vm`, `filesys` 디렉터리가 있는 Pintos 프로젝트 루트에서 아래 명령을 실행하세요.

```bash
TEAM=YOUR_TEAM_NUMBER make archive
```

예:

```bash
TEAM=5 make archive
```

그러면 같은 디렉터리에 `team5.tar.gz`가 생성됩니다. 이 파일을 제출하세요.

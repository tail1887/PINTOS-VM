# Introduction

## 개요

KAIST CS330의 Pintos 프로젝트에 오신 것을 환영합니다.  
이번 학기에는 KAIST CS330 수업에 맞게 특별히 수정된 Pintos 버전을 사용합니다.

## 먼저 알아둘 점

- 요구사항이 원본(vanilla) Pintos와 일부 다릅니다.
- 따라서 **이 매뉴얼의 지시를 최우선으로** 따라야 합니다.
- 이전에 진행한 본인 Pintos 프로젝트를 참고할 수는 있지만, 업데이트된 요구사항에 맞춰 일부 항목은 다시 구현해야 합니다.

## 매뉴얼 동기화 관련 안내

변경 사항에 따라 담당 교수진이 매뉴얼을 수정했습니다.  
가능한 한 코드와 문서를 동기화했지만, 일부 구간은 완전히 일치하지 않을 수 있습니다.  
불일치 구간을 발견하면 교수진에게 알려주세요.

> 반드시 `Legal and Ethical Issues` 섹션을 읽어야 합니다.

## Pintos 프로젝트 소개

`pintos-kaist`는 x86-64 아키텍처를 위한 간단한 운영체제 프레임워크입니다.  
이 프로젝트는 Stanford University의 Pintos 프로젝트를 포크해 시작했습니다.  
앞으로는 `pintos-kaist` 대신 `Pintos`라는 이름을 사용합니다.

Pintos는 다음 기능을 매우 단순한 형태로 제공합니다.

- 커널 스레드
- 사용자 프로그램 로딩/실행
- 파일시스템

프로젝트에서는 위 세 영역을 강화하고, 추가로 가상 메모리 구현도 수행합니다.

## 왜 시뮬레이터를 사용하는가

이론적으로 Pintos는 일반 x86-64 PC에서도 실행 가능합니다.  
하지만 모든 학생에게 전용 PC를 제공하는 것은 현실적으로 어렵기 때문에, 시스템 시뮬레이터를 사용합니다.

시스템 시뮬레이터는 x86-64 CPU와 주변 장치를 충분히 정확하게 모사하여,  
수정되지 않은 운영체제와 소프트웨어가 그대로 동작할 수 있게 해 주는 프로그램입니다.  
수업에서는 QEMU 시뮬레이터를 사용합니다.

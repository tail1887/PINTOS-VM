# Argument Passing

원문: [Argument Passing](https://casys-kaist.github.io/pintos-kaist/project2/argument_passing.html)

## Argument Passing

**`process_exec()`에서 사용자 프로그램 인자를 설정합니다.**

### x86-64 호출 규약

이 절은 64비트 x86-64 유닉스 구현에서 일반 함수 호출에 쓰이는 규약의 중요한 요점만 요약합니다. 자세한 내용은 생략했으며, 더 알고 싶다면 **System V AMD64 ABI**를 참고하세요.

호출 규약은 대략 다음과 같습니다.

1. 사용자 수준 애플리케이션은 정수 인자를 `%rdi`, `%rsi`, `%rdx`, `%rcx`, `%r8`, `%r9` 순으로 넘깁니다.
2. 호출자는 다음에 실행할 명령의 주소(복귀 주소)를 스택에 넣고, 피호출자의 첫 번째 명령으로 점프합니다. x86-64의 단일 명령 `CALL`이 이 두 동작을 모두 수행합니다.
3. 피호출자가 실행됩니다.
4. 피호출자에 반환값이 있으면 레지스터 RAX에 넣습니다.
5. 피호출자는 스택에서 복귀 주소를 꺼내 그 주소로 점프하여 돌아가며, x86-64에서는 `RET` 명령을 사용합니다.

정수 인자 세 개를 받는 함수 `f()`를 생각해 보세요. 아래 다이어그램은 위 3번 단계 시작 시점에 피호출자가 보는 스택 프레임과 레지스터 상태를 나타냅니다. `f()`가 `f(1, 2, 3)`으로 호출되었다고 가정합니다. 초기 스택 주소는 임의입니다.

```
                             +----------------+
stack pointer --> 0x4747fe70 | return address |
                             +----------------+
RDI: 0x0000000000000001 | RSI: 0x0000000000000002 | RDX: 0x0000000000000003
```

### 프로그램 시작 세부 사항

사용자 프로그램용 Pintos C 라이브러리는 `lib/user/entry.c`의 `_start()`를 진입점으로 둡니다. 이 함수는 `main()`을 감싸는 래퍼이며, `main()`이 반환되면 `exit()`를 호출합니다.

```c
void
_start (int argc, char *argv[]) {
    exit (main (argc, argv));
}
```

커널은 사용자 프로그램이 실행을 시작하기 **전에** 최초 함수에 넘길 인자를 레지스터에 올려 두어야 합니다. 인자는 일반 호출 규약과 같은 방식으로 전달됩니다.

예시 명령 `/bin/ls -l foo bar`의 인자를 어떻게 다룰지 생각해 보세요.

1. 명령을 단어로 나눕니다: `/bin/ls`, `-l`, `foo`, `bar`.
2. 단어들을 스택 **맨 위**에 둡니다. 순서는 중요하지 않습니다. 포인터로 참조할 것이기 때문입니다.
3. 각 문자열 주소와 널 포인터 센티널을 스택에 **오른쪽에서 왼쪽** 순으로 넣습니다. 이것들이 `argv`의 원소입니다. 널 포인터 센티널은 C 표준에서 요구하는 대로 `argv[argc]`가 널 포인터가 되게 합니다. 이렇게 하면 `argv[0]`이 가장 낮은 가상 주소에 오게 됩니다. 워드 정렬 접근이 비정렬보다 빠르므로, 성능을 위해 첫 `push` 전에 스택 포인터를 8의 배수로 내림 정렬하는 것이 좋습니다.
4. `%rsi`가 `argv`(`argv[0]`의 주소)를 가리키게 하고, `%rdi`에 `argc`를 둡니다.
5. 마지막으로 가짜 "복귀 주소"를 넣습니다. 진입 함수는 실제로는 돌아가지 않지만, 스택 프레임 구조는 다른 함수와 같아야 합니다.

아래 표는 사용자 프로그램이 막 시작되기 직전의 스택과 관련 레지스터 상태입니다. 스택은 아래로 자랍니다.

| Address    | Name             | Data         | Type         |
| ---------- | ---------------- | ------------ | ------------ |
| 0x4747fffc | argv\[3\]\[...\] | 'bar\\0'     | char\[4\]    |
| 0x4747fff8 | argv\[2\]\[...\] | 'foo\\0'     | char\[4\]    |
| 0x4747fff5 | argv\[1\]\[...\] | '-l\\0'      | char\[3\]    |
| 0x4747ffed | argv\[0\]\[...\] | '/bin/ls\\0' | char\[8\]    |
| 0x4747ffe8 | word-align       | 0            | uint8\_t\[\] |
| 0x4747ffe0 | argv\[4\]        | 0            | char \*      |
| 0x4747ffd8 | argv\[3\]        | 0x4747fffc   | char \*      |
| 0x4747ffd0 | argv\[2\]        | 0x4747fff8   | char \*      |
| 0x4747ffc8 | argv\[1\]        | 0x4747fff5   | char \*      |
| 0x4747ffc0 | argv\[0\]        | 0x4747ffed   | char \*      |
| 0x4747ffb8 | return address   | 0            | void (\*) () |

> RDI: 4 | RSI: 0x4747ffc0

이 예에서는 스택 포인터가 `0x4747ffb8`로 초기화됩니다. 위와 같이, 코드는 `include/threads/vaddr.h`에 정의된 `USER_STACK`에서 스택을 시작해야 합니다.

인자 전달 코드를 디버깅할 때 `<stdio.h>`에 선언된 비표준 `hex_dump()` 함수가 유용할 수 있습니다.

### 인자 전달 구현

현재 `process_exec()`는 새 프로세스에 인자를 넘기지 않습니다. 인자로 프로그램 파일 이름만 받는 대신, **공백으로 단어를 나누도록** `process_exec()`를 확장해 이 기능을 구현하세요. 첫 번째 단어는 프로그램 이름, 두 번째는 첫 번째 인자, 이하 동일합니다. 즉 `process_exec("grep foo bar")`는 `grep`을 실행하고 인자 두 개 `foo`, `bar`를 넘깁니다.

한 줄 안에서 **연속된 공백은 하나의 공백과 동일**하므로 `process_exec("grep  foo   bar")`는 위 예와 같아야 합니다. 명령줄 인자 길이에 **합리적인 상한**을 둘 수 있습니다. 예를 들어 한 페이지(4kB) 안에 들어가는 인자만 허용하는 식입니다. (pintos 유틸리티가 커널에 넘길 수 있는 명령줄 인자에는 **별도로 128바이트 제한**이 있습니다.)

인자 문자열 파싱 방식은 자유입니다. 막막하면 `strtok_r()`을 보세요. 프로토타입은 `include/lib/string.h`에 있고, `lib/string.c`에 주석이 상세합니다. `man strtok_r`로 매뉴얼을 볼 수도 있습니다.

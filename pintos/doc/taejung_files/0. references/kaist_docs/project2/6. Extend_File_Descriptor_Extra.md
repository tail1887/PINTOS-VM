## Extend File Descriptor (Extra)

**리눅스처럼 표준 입력·표준 출력을 닫는 것과 `dup2` 시스템 콜을 지원하세요.**

현재 Pintos 구현에서는 `stdin`과 `stdout`의 fd를 닫는 것이 금지되어 있습니다. 추가 점수 요구사항에서는 **리눅스와 같이** 사용자가 `stdin`과 `stdout`을 닫을 수 있게 해야 합니다. 즉, `stdin`을 닫으면 그 프로세스는 더 이상 입력을 읽을 수 없어야 하고, `stdout`을 닫으면 더 이상 출력을 할 수 없어야 합니다.

그다음 **`dup2` 시스템 콜**을 구현합니다.

---

```c
int dup2(int oldfd, int newfd);
```

> `dup2()` 시스템 콜은 파일 디스크립터 `oldfd`의 복사본을 `newfd`에 지정된 번호로 만들고, 성공 시 `newfd`를 반환합니다. `newfd`가 이전에 열려 있었다면, **재사용되기 전에 조용히 닫힙니다.**

> 다음을 유의하세요.

> - `oldfd`가 유효한 파일 디스크립터가 **아니면** 호출은 실패하고(`-1` 반환) `newfd`는 닫히지 **않습니다.**
> - `oldfd`가 유효하고 `newfd`가 `oldfd`와 **같은 값이면** `dup2()`는 아무 일도 하지 않고 `newfd`를 반환합니다.

> 이 시스템 콜이 성공적으로 돌아온 뒤에는 이전 fd와 새 fd를 **서로 바꿔 써도** 됩니다. 서로 **다른 파일 디스크립터**이지만 **같은 열린 파일 설명(open file description)**을 가리키므로 **파일 오프셋**과 **상태 플래그**를 공유합니다. 예를 들어 한쪽에서 `seek`로 오프셋을 바꾸면 다른 쪽에도 반영됩니다.

`dup`으로 만든 파일 디스크립터는 **`fork` 이후에도** 의미가 유지되어야 합니다.
